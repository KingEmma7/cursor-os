# Code of conduct

Cursor OS is a small open-source project. Contributions and discussion should stay constructive, specific, and respectful.

## Expected behavior

- Assume good intent, but stay precise about technical trade-offs.
- Critique ideas, code, and documentation — not people.
- Keep issues and pull requests focused on concrete behavior.
- Be welcoming to people trying Cursor OS in different kinds of projects.
- Respect maintainers' time by providing reproduction steps, context, and verification output.

## Unacceptable behavior

- Harassment, abuse, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeated off-topic comments after a maintainer asks to refocus.
- Bad-faith contributions intended to degrade the project or mislead users.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or pull requests; or block participants whose behavior is harmful to the project or community.

If you need to report a conduct issue, contact the maintainer privately through the contact method listed on the maintainer's GitHub profile.
